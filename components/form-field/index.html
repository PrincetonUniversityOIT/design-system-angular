Form Field

A Form Field is a composite form construct that encapsulates a form input along with its corresponding label, required/optional indicator, field-specific assistive content, and field-specific error messages. 

<form-field label="Field 1" for="field1" [messageConfig]="{'required': '\'Field 1\' is required.'}" required="required">
  <input type="text" formInput id="field1" class="jazz-input" placeholder="Enter Field 1 Here"
         [class.jazz-input--error]="form.get('field1').invalid && (form.get('field1').dirty || form.get('field1').touched)"
         formControlName="field1" required />
</form-field>

Best Practices

  • Clearly label each form input.
  • Provide field-specific assistive content to clarify how the field should be used/completed.
  • Clearly label fields as required/optional. If most of the fields are required, indicate which are optional. If most of the fields are optional, indicate which are required.
  • Capitalize the first word of the label and any proper names as they would normally be capitalized.

The Basics

  <input type="text" formInput id="field1" class="jazz-input" placeholder="Enter Field 1 Here"

         formControlName="field1" required />
</form-field>

Detailed Specification

Design

  • A Form Field is composed of the label, the input control and the error message component.
  • The input control contained within the form field tags can be any simple reactive form control.
  • The form field is designed to contain only one input control at a time.

Code

  • To customize the text for the label, set the label property to desired text.
  • The property for needs to be set in order to link the label to the input control properly for accessibility.
  • The property required can be set in order to turn on the required validation and to show the red asterisk next to the label to indicate that the field is required.
  • In order to override or specify messages for custom validation errors the messageConfig property needs to be set containing an object with the validator key and the error message that you want to show for the validation error. 
    [messageConfig]="{'required': '\'Field 1\' is required.'}"
  • The input control needs to have the formInput directive so that the component knows which control to check for errors and to check if the component needs to be disabled.
<form-field label="Field 1" for="field1" [messageConfig]="{'required': '\'Field 1\' is required.'}" required="required">
      <input type="text" formInput id="field1" class="jazz-input" placeholder="Enter Field 1 Here"
             [class.jazz-input--error]="form.get('field1').invalid && (form.get('field1').dirty || form.get('field1').touched)"
             formControlName="field1" required />
    </form-field>

Examples

Here are some examples on how to use the form field component with various form controls.

Input Text

<form-field label="Field 1" for="field1" [messageConfig]="{'required': '\'Field 1\' is required.'}" required="required">
  <input type="text" formInput id="field1" class="jazz-input" placeholder="Enter Field 1 Here"
         [class.jazz-input--error]="form.get('field1').invalid && (form.get('field1').dirty || form.get('field1').touched)"
         formControlName="field1" required />
</form-field>

Text Area

<form-field label="Field 2" for="field2" required="required">
    <textarea class="jazz-input full-width" id="field2" formInput type="text"
        [class.jazz-input--error]="form.get('field2').invalid && (form.get('field2').dirty || form.get('field2').touched)"
        formControlName="field2"
    ></textarea>
  </form-field>

Dropdown

<form-field label="Field 3" for="field3" required="required">
  <select id="field3" class="jazz-input" formInput formControlName="field3"
          [class.jazz-input--error]="form.get('field3').invalid && (form.get('field3').dirty || form.get('field3').touched)">
    <option></option>
    <option>Option 1</option>
    <option>Option 2</option>
    <option disabled>Option 3</option>
    <option>Option 4</option>
  </select>
</form-field>

Radio Button

  <input formInput type="radio" name="field4" formControlName="field4" value=1 checked><span>Text 1</span>
  <input formInput type="radio" name="field4" formControlName="field4" value=0><span>Text 2</span>
</form-field>

Check Box

  <input formInput type="checkbox" id="checkbox-2" value="value-2" formControlName="field5"><span>Input 2</span>
  <input formInput type="checkbox" id="checkbox-3" value="value-2" formControlName="field5"><span>Input 3</span>

States

The states described below apply to each Accordion Item, not to the Accordion as a whole.

Default

The Default state of a Form Field is Empty and not in the Error state.

Error (Warning/Invalid)

The Error state of a Form Field indicates that the Form Field’s child input values are not value and optionally provides an error message that indicates the specific problem with the field.

Correct

The Correct state indicates that the Form Field’s child input values are correct/valid. In most cases, this state would not need to be represented, but it may be useful in cases where achieving the correct value is difficult.

Required

The Required state indicates that a Form Field is required, meaning that a value must be specified for the inputs in that Form Field before the value can be accepted.

Empty

There are a few different Empty state situations for a Form Field that need to be considered:

One (Single, Only)

The One state for a Text Input indicates that a value has been provided for the input. Any placeholders are usually hidden once a value has been provided.

Overflow

The Overflow state for a Text Input would be when the value specified for the component exceeds the available space to display that value. By default, HTML Form inputs will truncate the display of any content that exceeds the available width, while preserving the underlying value.

Theme Variables

Colors

To override color variables with colors from the Design System color palette you can specify color hex values or html colors. If you want to use colors from the Design System color palette, you can also specify the following value format for the color you want to use for the variable: 

var(--jazz-gray);

Fonts

To override the font variables you can specify the font you want to use as you would typically put in the css. Here is a quick example: 

'Lora', serif
